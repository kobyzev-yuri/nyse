Как считается новостной импульс (L3 → L5 → L6)

L3 — только «сырые» скоры по статьям, без LLM-агрегатора: для каждой статьи cheap_sentiment ∈ [-1,1] (FinBERT / API / price_pattern); затем draft_impulse по каналам INC / REG / POL с экспоненциальным затуханием по времени. Из этого строится один скаляр draft_bias (блок ⑤).

В этом прогоне draft_bias = +0.2294. Он идёт в гейт L4 (решение SKIP / LITE / FULL и текст причины в блоке ⑥), но не подставляется в формулу Fusion как «новости 30%».

L5 — если сработал structured LLM (режим FULL в этом пайплайне): модель по каждой строке батча возвращает поля sentiment, impact_strength, relevance, time_horizon, confidence. Функция aggregate_news_signals (pipeline/news_signal_aggregator.py) считает взвешенное среднее:

  • w = w(relevance) × w(impact) × w(horizon) × max(confidence, 0.05) — те же веса, что в pystockinvest;
  • bias в таблице ниже = Σ(sentiment × w) / Σw;
  • confidence в таблице = Σ(confidenceстроки × w) / Σw.

Строки summary вроде «Aggregated news bias is 0.62…» — человекочитаемое повторение этих двух чисел, а не отдельный расчёт.

Сейчас: LLMMode = FULL; строк в LLM-агрегате: 12; AggregatedNewsSignal.bias = +0.5694.

L6 — Fusion (блок ②): в итоговый final_bias новости входят только как W_NEWS × news.bias → вклад +0.1708 (= W_NEWS=0.3 × news.bias=+0.5694) (константа W_NEWS в pipeline/trade_builder.py).

При SKIP или LITE вызов structured LLM в run_news_signal_pipeline не делается: возвращается нейтральный агрегат (bias 0), поэтому вклад News в Fusion = 0, даже если draft_bias был сильным.
