# Как работает пайплайн новостей — своими словами

Это конвейер из шести уровней, где данные последовательно конкретизируются — от «сырых заголовков» до «скалярного числа для торгового решения». Экономия вычислений — ключевой принцип: дорогой LLM вызывается только когда дешёвые уровни уже нашли сигнал.

---

## Уровень 0 — Сбор: статья как объект данных

Несколько провайдеров (Yahoo, Marketaux, NewsAPI, Alpha Vantage, RSS) каждый умеет что-то своё:

- Yahoo — бесплатный baseline, заголовки по тикеру
- Marketaux — уже отдаёт числовой `raw_sentiment` для каждой упомянутой компании
- RSS — макро-ленты ЦБ без привязки к тикеру

Все адаптеры нормализуют ответ в один тип `NewsArticle`:

```
title:           "NBIS Jumped 15% on Its Meta Deal"
timestamp:       2026-04-05T14:30Z
ticker:          NBIS
provider_id:     "marketaux"
raw_sentiment:   +0.72        ← от Marketaux, если есть
cheap_sentiment: None         ← заполнится на уровне 2
```

Кэш (`raw|v1|marketaux|NBIS|...`) сохраняет ответ на 15 минут — второй вызов в тесте не бьёт API.

---

## Уровень 1 — Канал воздействия: к какому типу относится новость

Без LLM, чистые регулярные выражения. Каждая статья получает тег канала:

- `REGIME` — если в тексте есть `war`, `sanctions`, `invasion`, `embargo`, `NATO` и т.д.
- `POLICY_RATES` — если есть `Fed`, `FOMC`, `rate hike`, `ECB`, `bps`
- `INCREMENTAL` — всё остальное (отчётность, аналитики, отраслевые новости)

Приоритет: REGIME > POLICY_RATES > INCREMENTAL. Уверенность — грубая (`0.85` / `0.80`), не ML.

**Зачем это нужно:** три типа новостей влияют на рынок очень по-разному. Новость о санкциях и новость о квартальной выручке NBIS — принципиально разные сигналы, их нельзя усреднять в одну кучу.

---

## Уровень 2 — `cheap_sentiment`: скаляр [−1, 1] на каждую статью

Это самый важный «атом» — единое число, отвечающее на вопрос «насколько статья бычья или медвежья».

Три источника с приоритетом:

1. **`raw_sentiment` от провайдера** (Marketaux, AV) — берётся напрямую, обрезается в [−1, 1]. Если есть — дальше не идём.
2. **FinBERT** — локальная HuggingFace-модель, классифицирует текст `title + summary` на POSITIVE/NEGATIVE/NEUTRAL и переводит в float. Работает без интернета, кэш по `sha256(model_name + text)` на сутки.
3. **`price_pattern_boost`** — если FinBERT дал слабый сигнал, но в заголовке явное "jumped 15%" или "sinks 7%" — boost работает как floor. Пример: "NBIS Jumped 15%..." → boost = +0.80. Если FinBERT выдал +0.1 (слишком осторожно), а boost +0.80 > 0.1 → итог +0.80.

Масштаб boost: `≥20% → ±1.0`, `≥10% → ±0.8`, `≥5% → ±0.6`, `≥2% → ±0.4`, иначе `±0.2`.

После уровня 2: у каждой статьи есть `cheap_sentiment` и канал.

---

## Уровень 3 — `DraftImpulse`: черновой импульс по окну времени

Агрегация всех статей за последние N часов, но **каналы строго раздельны**:

```
DraftImpulse:
  draft_bias_incremental  = взвешенное среднее cheap_sentiment  [INCREMENTAL]
  regime_stress           = взвешенное среднее |cheap_sentiment| [REGIME]
  policy_stress           = взвешенное среднее |cheap_sentiment| [POLICY_RATES]
```

**Затухание по времени:** статья, вышедшая 12 часов назад, весит в 2 раза меньше только что вышедшей. Формула: `w = exp(−ln(2) / half_life × age_hours)`.

Для REGIME и POLICY важна **сила** (абсолютное значение), а не направление — потому что одна статья про санкции говорит «рост напряжённости», другая «переговоры», но обе поднимают стресс. Их среднее по знаку давало бы 0, что неверно.

Скалярный прокси для гейта:
```
scalar = draft_bias_incremental − 0.15 × regime_stress − 0.1 × policy_stress
```

Если фон напряжён (большой `regime_stress`), итоговый скаляр снижается — даже позитивные тикерные новости становятся менее убедительными.

---

## Уровень 4 — Гейт: нужен ли LLM вообще?

Это ключевой узел экономии. По скаляру из уровня 3 и контексту принимается решение `SKIP / LITE / FULL`:

```
calendar_high_soon?               → FULL  (скоро CPI/FOMC/NFP — всегда вызываем)
regime_present И confidence ≥ T2? → FULL  (геополитика высокой уверенности)
|bias| ≥ T1 × 2.0?               → FULL  (сильный сигнал — приоритет над лимитом)
|bias| < T1 И нет REGIME?         → SKIP  (тихий день — LLM не нужен)
article_count > N?                → LITE  (много статей — урезанный батч)
иначе                             → LITE
```

Пороги T1, T2, N зависят от профиля тикера:

| Профиль | T1 | N | Для |
|---------|----|---|-----|
| `PROFILE_GAME5M` | 0.12 | 8 | NBIS, MU, SNDK, LITE, CIEN, ASML |
| `PROFILE_CONTEXT` | 0.20 | 15 | MSFT, META, NVDA, AMZN |

`calendar_high_soon` — отдельный модуль `calendar_context.py` проверяет, есть ли в ближайшие ±30 минут событие HIGH-важности (CPI, NFP, заседание ФРС).

На `SKIP` пайплайн заканчивается — в качестве новостного импульса используется `draft_bias` из уровня 3 с пониженной confidence.

---

## Уровень 5 — LLM: структурированный сигнал Kerima-стиля

Вызывается только при LITE или FULL. На вход — список статей (при LITE — топ-N по `|cheap_sentiment|`). LLM просят вернуть по каждой статье:

```json
{
  "article_index": 2,
  "sentiment":       0.7,
  "impact_strength": "high",
  "relevance":       "primary",
  "surprise":        "significant",
  "time_horizon":    "1-3d",
  "confidence":      0.85
}
```

Это `NewsSignal` — каждая статья получила смысловую оценку, а не просто «бычья/медвежья».

Ответ кэшируется по `sha256(messages + model + prompt_version)` на 60 минут — повторный вызов с теми же статьями бесплатен.

Затем агрегация в `AggregatedNewsSignal`:

```
bias = Σ( sentiment × relevance_w × impact_w × horizon_w × confidence ) / Σ(weights)
```

Веса: `relevance` (mention=0.4, related=0.7, primary=1.0) × `impact` (low=0.4, moderate=0.7, high=1.0) × `horizon` (intraday=0.8, 1-3d=1.0, 3-7d=0.6, long=0.3).

Логика весов: упоминание тикера вскользь в статье о другой компании (`relevance=mention`) почти не влияет; короткий горизонт 1-3d весит сильнее длинного; уверенность LLM дополнительно масштабирует вклад каждой статьи.

**Итог уровня 5:** `AggregatedNewsSignal(bias=+0.61, confidence=0.78, summary=[...], items=[...])`.

## Уровень 6 — Слияние с техническим сигналом (в плане)

Технический сигнал `TechnicalSignal` — это то, что делает Kerima: RSI, ATR, SMA, breakout, Finviz-метрики, свечи → `TechnicalSignal(bias, confidence, tradeability_score, ...)`.

Оба сигнала имеют одинаковый масштаб `bias ∈ [−1, 1]` и `confidence ∈ [0, 1]` — именно поэтому домен `NewsArticle`, `NewsSignal`, `AggregatedNewsSignal` согласован с `pystockinvest/agent/models.py`.

Слияние в `FusedSignal` (TradeBuilder-логика):

```
fused_bias = w_tech × tech.bias + w_news × news.bias + w_cal × cal.bias
```

При `REGIME`-фоне вводится `regime_overhang` — понижает итоговую `confidence` сделки, даже если и техника, и новости говорят BUY. Логика: геополитический шок обнуляет предсказательную силу локальных сигналов до выяснения направления.

**Итоговый вывод:** `domain.Trade` с конкретной `Position` — цена входа, стоп, цель. Подробный пример — в следующем разделе.

## Всё одной схемой

```
Yahoo / Marketaux / RSS / NewsAPI
         │ NewsArticle[]
         ▼
[1] classify_channel()          → INCREMENTAL | REGIME | POLICY_RATES
         │
         ▼
[2] resolve_cheap_sentiment()   → float [−1,1] на статью
    (raw_sentiment → FinBERT → price_pattern_boost как floor)
         │
         ▼
[3] draft_impulse()             → DraftImpulse
    (exp-затухание, каналы раздельно)
    + CalendarEvent HIGH?
         │
         ▼
[4] decide_llm_mode()           → SKIP / LITE / FULL
    (T1, T2, N, calendar_high_soon)
         │
    ┌────┴─────────────┐
  SKIP               LITE/FULL
  draft_bias              │
  как proxy               ▼
                 [5] LLM (Kerima) → NewsSignal[] → AggregatedNewsSignal
                                    (bias, confidence, summary)
                                          │
                                          ▼
                                [6] TradeBuilder (план)
                                tech.bias + news.bias + cal.bias
                                          │
                                          ▼
                                FusedSignal(side, bias, confidence)
```

Самое важное — LLM используется экономно. На тихом дне большинство тикеров получают `SKIP` и работают на дешёвом `DraftImpulse`. LLM включается только когда что-то действительно произошло: рынок двигается, есть геополитика или скоро выходят важные данные.

## Пример сквозного расчёта: NBIS, сильная новость

Разбираем конкретный случай — тикер NBIS, вышедшая новость "Jumped 15% on Its Meta Deal".

### Что собрал уровень 0

Marketaux отдал статью с готовым `raw_sentiment = +0.72`. Yahoo добавил ещё 4 заголовка по тому же тикеру.

### Уровень 1 → 2: канал и сентимент

У всех пяти статей — канал `INCREMENTAL` (нет слов `war`, `sanctions`, `Fed`).

Для главной статьи:
- `raw_sentiment = +0.72` (Marketaux) → `cheap_sentiment = +0.72` напрямую
- FinBERT не нужен (приоритет 1 отработал)

Для остальных четырёх — FinBERT: оценки слабые (+0.10, +0.05, −0.10, +0.15). Для одного из них заголовок содержал "rose 3%" → `price_pattern_boost = +0.40`; FinBERT давал +0.10, boost перекрыл → итог +0.40.

### Уровень 3: DraftImpulse

Все пять — канал `INCREMENTAL`, `regime_stress = 0.0`, `policy_stress = 0.0`.

```
Статья              age     weight   cheap_sentiment
"Jumped 15%…"      0.5 ч    0.97     +0.72
"rose 3%…"         2.0 ч    0.89     +0.40
прочие (3 шт)      4–6 ч    ~0.75    ~+0.05

draft_bias_incremental ≈ (0.97×0.72 + 0.89×0.40 + 3×0.75×0.05) / Σw
                       ≈ (0.699 + 0.356 + 0.113) / 3.36 ≈ +0.35
```

Скалярный прокси = `0.35 − 0.15×0 − 0.1×0 = +0.35`.

### Уровень 4: гейт

Профиль `PROFILE_GAME5M`: T1 = 0.12, N = 8.

```
calendar_high_soon?  → нет (сейчас не CPI/NFP)
regime_present?      → нет (regime_stress = 0.0)
|bias| ≥ T1×2 = 0.24? → 0.35 ≥ 0.24 → ДА → FULL
```

LLM вызывается в полном режиме на все 5 статей.

### Уровень 5: LLM → AggregatedNewsSignal

LLM оценил каждую статью. Главная получила:

```json
{ "sentiment": 0.75, "impact_strength": "high",
  "relevance": "primary", "surprise": "significant",
  "time_horizon": "1-3d", "confidence": 0.85 }
```

Агрегация (формула `relevance × impact × horizon × confidence`):

```
главная:   w = 1.0 × 1.0 × 1.0 × 0.85 = 0.850
остальные: w ≈ 0.7 × 0.4 × 0.8 × 0.55 ≈ 0.123 × 4 = 0.492

bias = (0.850×0.75 + 0.492×~0.15) / (0.850 + 0.492)
     ≈ (0.638 + 0.074) / 1.342 ≈ +0.53
```

`AggregatedNewsSignal(bias=+0.53, confidence=0.76)`.

### Уровень 6: TradeBuilder

```
final_bias = 0.55 × tech.bias  +  0.30 × news.bias  +  0.15 × cal.bias
           = 0.55 × 0.68       +  0.30 × 0.53       +  0.15 × 0.10
           = 0.374 + 0.159 + 0.015  =  +0.548
```

Уверенность (с бонусом за согласованность и штрафом за calendar_risk=0.20):

```
agreement_bonus = 0.548 × 0.15 = 0.082
raw    = 0.50×0.70 + 0.30×0.76 + 0.20×0.65 + 0.082 = 0.792
conf   = 0.792 × (1 − 0.35×0.20) = 0.792 × 0.93 = 0.737
```

Тип ордера:

```
breakout_score (0.80) ≥ mean_reversion_score (0.30) → True
exhaustion_score (0.20) < 0.65                      → True
|news.bias| = 0.53 > 0.40  (urgent_news)            → True
→ entry_type = MARKET
```

Уровни по ATR (`price = 17.40`, `ATR = 0.85`, `volatility_regime = 0.45`):

```
entry      = 17.40   (рыночный ордер)
stop_loss  = 17.40 − (1.0 + 0.4×0.45) × 0.85 = 17.40 − 1.18×0.85 = 16.40
take_profit = 17.40 + (1.8 + 0.5×0.55) × 0.85 = 17.40 + 2.07×0.85 = 19.16
```

### Итоговый `Trade`

```python
Trade(
  ticker:     NBIS,
  entry_type: MARKET,
  position: Position(
    side:        LONG,
    entry:       17.40,
    stop_loss:   16.40,   # −$1.00 от входа (−5.7%)
    take_profit: 19.16,   # +$1.76 от входа (+10.1%)
    confidence:  0.737,
  ),
  news_summary: [
    "Aggregated news bias is 0.53 on a -1 to 1 scale.",
    "News confidence is 0.76.",
  ],
)
```

**Risk/Reward = 1.76 / 1.00 = 1.76** — в допустимом диапазоне для интрадей.

### Где новостной импульс повлиял критически

Без новостного сигнала (при `SKIP`):
- `news.bias = 0.0` → `final_bias = 0.374 + 0 + 0.015 = 0.389`
- `conf` немного ниже (нет новостного 30%)
- Главное: `urgent_news = False` → `entry_type = LIMIT` вместо `MARKET` → ждём откат, который при +15%-новости может не прийти

Именно поэтому пайплайн новостей важен не только для «направления», но и для **скорости входа**.
